**Verified facts (read during planning):**
- Notification scheduling (flutter_local_notifications 21.0.0): `zonedSchedule({required int id, required tz.TZDateTime scheduledDate, required NotificationDetails notificationDetails, required AndroidScheduleMode androidScheduleMode, String? title, String? body, String? payload, DateTimeComponents? matchDateTimeComponents})` — all named. Needs `timezone` init (`tz_data.initializeTimeZones()`); a RELATIVE one-shot can schedule against `tz.TZDateTime.now(tz.UTC).add(...)` — no `flutter_timezone`, no `setLocalLocation`. `AndroidScheduleMode.exactAllowWhileIdle`. Manifest: `<uses-permission android:name="android.permission.USE_EXACT_ALARM"/>` (auto-granted for timer apps, no prompt) + `<receiver android:exported="false" android:name="com.dexterous.flutterlocalnotifications.ScheduledNotificationReceiver"/>` (scheduling) + `<receiver android:exported="false" android:name="com.dexterous.flutterlocalnotifications.ActionBroadcastReceiver"/>` (action buttons). No boot receiver needed (one-shot, no reboot persistence). No foreground service.
- Action chip: `AndroidNotificationAction('add_30s', '+30s', showsUserInterface: false, cancelNotification: false)` in `AndroidNotificationDetails(actions: [...])`. `showsUserInterface:false` routes the tap to the BACKGROUND isolate handler; `cancelNotification:false` keeps it shown.
- Background handler: top-level `@pragma('vm:entry-point') void f(NotificationResponse r)`, passed as `onDidReceiveBackgroundNotificationResponse:` to `initialize`. Runs in a FRESH isolate — must create a new `FlutterLocalNotificationsPlugin()` + call `initialize()` AND `initializeTimeZones()` inside it before scheduling; can read `SharedPreferences`. Distinguish via `r.actionId == 'add_30s'`. The function must exist/compile on all platforms (referenced by name) but only runs on Android.
- Current files: `template_absorb.dart` (absorbOps signature with affectedSets/affectedItems/affectedSessionDayIds/existingIds/alreadyAbsorbed; exercise INSERT at :74-98), `volume_bars.dart` (maxVal at :41-44, fractions at :88-89), `session_manager.dart` (`_armRestExpiry` :66-76, `register`/`_onControllerChange` call `notifier?.showFor`), `workout_notification.dart` (init/showFor/cancel + pure `notificationPayloadFor`), `session_mini_bar.dart` (full-width pill, `_fmt`), `app_shell.dart` (mini-bar Builder at :161-179, Stack), `today_screen.dart` (hero at :199-214, `_buildSplitCard` :265, `_dayList`, padding `bottom: 96` :179), `active_session_screen.dart` (rest start `controller.startRest(b.exercise.compound ? 180 : 90)` :245, RestTimerCard :305-316, ticker :60), `settings_service.dart` (field+getter+setter+load pattern), `exercise_repository.dart` (`exerciseUpsertOp` :53-113, INSERT col list :66-70), `models.dart` (`Exercise` :49-105 + `fromRow` :84), `sync_upload.go` `applyExercise` (:261-349 PUT+PATCH), `schema.dart` exercises (:18-35, no rest col).
