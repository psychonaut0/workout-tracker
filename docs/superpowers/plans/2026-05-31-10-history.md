# History Screen Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Replace the History placeholder tab with the real read-only History screen: sessions grouped by ISO week, a 4-week summary (Sessions / PRs / Volume), and expandable session cards showing per-exercise top sets with PR markers.

**Architecture:** One new grouped per-session stats query (`SessionRepository.watchSessionStats`) feeds a `StreamBuilder`; the screen groups client-side by Monday-week (reusing `app/lib/util/dates.dart`) and computes the 4-week summary. Each card expands to lazily load its exercise blocks (`setsForSession` + `groupIntoBlocks`, already exist), resolving exercise name + compound from the catalog. **Read-only — no migration, no writes.** Mounts at `AppShell` IndexedStack index 2.

**Tech Stack:** Flutter 3.44 (`make -C app`), PowerSync. Reuse `context.tokens`/`WorkoutType`/`WIcons`/`AppRadius`, `UnitService` (`fromKg`/`fmtWt`/`uLabel`), `WCard`/`PRBadge`, `dates.dart` (`weekStart`/`isoDate`/`fmtDate`/`daysAgo`), `util/format.dart` (`fmtPlain`). Design spec: `docs/design_handoff_workout_tracker/design/app/screen-history.jsx`.

**Conventions:** Conventional Commits, subject only. Branch off `main`. The session card uses the stored `split_label` ("{name} · {focus}") directly — no day-template lookup.

- [ ] **Step 1: Add `HistorySessionRow` to `models.dart`:**
```dart
class HistorySessionRow {
  final String id, date;
  final String? splitLabel;
  final int? durationMin;
  final int exerciseCount;
  final int prCount;
  final double tonnageKg; // sum of non-warmup weight*reps
  HistorySessionRow({required this.id, required this.date, this.splitLabel, this.durationMin, required this.exerciseCount, required this.prCount, required this.tonnageKg});
  factory HistorySessionRow.fromRow(Map<String, dynamic> r) => HistorySessionRow(
        id: r['id'] as String,
        date: r['date'] as String,
        splitLabel: r['split_label'] as String?,
        durationMin: (r['duration_min'] as num?)?.toInt(),
        exerciseCount: (r['ex_count'] as num?)?.toInt() ?? 0,
        prCount: (r['pr_count'] as num?)?.toInt() ?? 0,
        tonnageKg: (r['tonnage'] as num?)?.toDouble() ?? 0,
      );
}
```

- [ ] **Step 2: Add to `SessionRepository`:**
```dart
Stream<List<HistorySessionRow>> watchSessionStats() => db.watch(
  '''SELECT se.id, se.date, se.split_label, se.duration_min,
            COUNT(DISTINCT s.exercise_id) AS ex_count,
            COUNT(DISTINCT CASE WHEN s.is_pr = 1 THEN s.exercise_id END) AS pr_count,
            COALESCE(SUM(CASE WHEN s.is_warmup = 0 THEN CAST(s.weight_kg AS REAL) * s.reps ELSE 0 END), 0) AS tonnage
       FROM sessions se LEFT JOIN sets s ON s.session_id = se.id
      GROUP BY se.id, se.date, se.split_label, se.duration_min
      ORDER BY se.date DESC, se.created_at ASC''',
).map((rs) => rs.map(HistorySessionRow.fromRow).toList());
```
(LEFT JOIN so a session with no sets still appears. `pr_count` counts distinct exercises with a PR. `tonnage` excludes warm-ups. `created_at ASC` in ORDER BY is harmless — date DESC dominates; the GROUP BY can't reference created_at in ORDER BY without aggregating, so use `MAX(se.created_at)` if the planner complains — but since created_at is functionally dependent on se.id which is grouped, SQLite allows it. If it errors, drop the created_at tiebreak.)

- [ ] **Step 3: Test** (`history_stats_test.dart`) — extract the week-grouping into a pure helper and test it instead of the DB query: add `Map<String, List<T>> groupByWeek<T>(List<T> items, String Function(T) dateOf)` to a small util (or in the screen file) that keys each item by `isoDate(weekStart(DateTime.parse(dateOf(item))))`.
